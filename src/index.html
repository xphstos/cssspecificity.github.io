CSS Specificity

Specificity is the means by which browsers decide which CSS property values are the most relevant to an element and, therefore, will be applied. Specificity is based on the matching rules which are composed of different sorts of CSS selectors.

Source: MDN Web Docs. (2018). Specificity.

Takeaway

As a rule of thumb, we use the universal selector (*) and element selectors to overwrite browser styles. Think of it as a reset the to the default styles given to elements by browsers. Remember, not all browsers have the same styles for the same elements.
Also it’s worth mentioning that while :not() counts as 0 the selector inside it, adds to the overall specificity!

Another important (pun intented) thing to notice is the use of !important in our styles. We must avoid them at all costs because it’s a clear indicator that something is going wrong. We might be able to get away with them in small projects but if we decide to scale it will come back and bite us sometime along the way.

We also need to use IDs sparingly, we usually do in order to target elements with JavaScript. The same goes for inline styles too. We must not add them by hand but it’s ok to add them or overide them via JavaScript.

We mostly talked about what type of selectors we shouldn’t use or when we should them. Let’s see what selectors we “should” use.

Again, I want to clarify that there are no absolute rules but if we want to keep our code clean and easy to maintain we can restrict ourselves to get there.

As we saw in the diagram above the green area is covering a small part of the elements selectors (e) and it goes all the way up to the IDs, which are ok to use but with caution. In terms of specificity this translates to levels 100, 200 or 300.

This means we need to be specific but not too much! We want to style our elements using only classes, along with pseudo-classes or attribute selectors and have no more than 2 levels of nesting. Again, this translates to 100 in specificity level.
In most cases this is not possible, so don’t go overboard with it.

Sounds difficult? Fear not because we have some tools for that. There are many CSS pre-processors like SCSS/SASS or Stylus but more importantly we have methodologies like BEM or SMACSS. Methodologies will help us structure our code in a way that stays clean, readable and easy to maintain with CSS especially with the help of preprocessors.
